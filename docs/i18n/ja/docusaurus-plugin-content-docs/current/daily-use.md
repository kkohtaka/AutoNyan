# 日常の使い方

毎日の流れはこうです。Drive に書類を置くと、AutoNyan が分類して整理し、通知
メールが届きます。インストールするものも、クリックする操作もありません。書類が
監視対象のフォルダに入れば、あとは自動で進みます。

## 書類を置く

[はじめに](./getting-started.md)で最初の書類を入れたときと同じ、共有された
**監視対象の Drive フォルダ**に書類を追加します。ファイルをドラッグするか、
いつもどおり Google Drive にアップロードしてください。

いくつか知っておくとよいことがあります。

- **すぐには処理されません。** AutoNyan は監視対象のフォルダをスケジュールに
  従って — 既定では**おおよそ 1 時間に 1 回** — スキャンします。書類を置いた
  直後ではありません。追加して数分たっても書類が監視対象のフォルダに残って
  いても、それは正常で、まだスキャンされていないだけです。
- **何件でも置けます。** 各書類は個別に処理されるので、まとめて置いても
  かまいません。
- **対応形式を使ってください。** AutoNyan は PDF とプレーンテキストのファイルを
  最後まで読み取ります。それ以外の形式は、見つかっても読み取れないことがあり、
  その場合は分類も整理もされません。詳しくは[対応ファイル形式](./supported-files.md)を
  参照してください。迷ったら、まず PDF として保存しておくと確実です。

## 分類と整理

スキャンで書類が拾われると、AutoNyan は中のテキストを読み取り、置き場所を判断
します。

1. **カテゴリを読み取ります。** カテゴリとは、Drive のカテゴリ領域にあなたが
   作成したフォルダ（たとえば `請求書`、`領収書`、`契約書`）そのものです。
   AutoNyan は毎回その時点の一覧を見るので、追加・名前変更したフォルダは次の
   書類から反映されます。
2. **最も合うものを選びます。** 書類のテキストをカテゴリ名と照合し、最も合う
   カテゴリを 1 つ選びます。あわせて、書類の短い要約と、そのカテゴリにどれだけ
   確信があるかを示す信頼度も生成します。
3. **書類をファイルします。** ファイルは監視対象のフォルダから該当するカテゴリ
   フォルダへ**移動**され、そのフォルダにある既存ファイルの命名スタイルに合わせ
   て、内容がわかる簡潔な名前に**リネーム**されます。うまく合うものがなければ、
   代わりに**未分類（Uncategorized）**フォルダへ元の名前のまま移します。これに
   より、AutoNyan が判断に迷っても書類が失われることはありません。

知っておくとよい点があります。

- **「未分類」は「失敗」ではなく「適切な一致がなかった」という意味です。**
  書類は読み取られてファイルされています。AutoNyan が十分に確信できるカテゴリを
  見つけられなかっただけです。特定の場所に入ると思っていた場合は、合致する
  カテゴリフォルダがあるか確認し、必要なら自分でファイルを移動してください。
- **ファイルはコピーではなく移動されます。** ファイル後、書類は監視対象の
  フォルダには残りません。カテゴリフォルダ（または未分類）で探してください。
- **リネームは控えめに行われます。** 新しい名前でも元のファイルの拡張子は
  そのまま維持されます。より良い名前に十分な確信が持てない場合、ファイルは
  元の名前のままになります。未分類フォルダに入った書類がリネームされることは
  ありません。
- **カテゴリはあなたが管理します。** 新しいカテゴリはフォルダを作って追加し、
  不要になったら削除します。AutoNyan が勝手にカテゴリを作ることはありません。

## 通知メール

AutoNyan は書類の処理を終えると、**通知メール**を送ります。わざわざ確認しに
行かなくても、何が起きたかがわかります。メールは書類 1 件につき 1 通です。

:::note

通知メールは現在、このドキュメントを読んでいる言語にかかわらず、**日本語**で
送信されます。

:::

**誰に届くか。** 処理成功の通知は、**移動先のカテゴリフォルダ**を共有している
人に届きます。つまり、あるカテゴリフォルダを見られる人には、新しい書類がそこに
入ったことが知らされます。処理失敗の通知は、該当するフォルダの**オーナー**に
届きます。

**書類が正常に処理されたとき**、メールには次の内容が記載されます。

- 処理された**ファイル名**（リネーム前の元の名前）、
- ファイルされた**カテゴリ**（合うものがなければ*未分類*）、
- そのカテゴリに対する**信頼度**、
- 選択の**理由**、
- 書類内容の短い**要約**。

書類が思ったとおりの場所にファイルされたかを確認するには、たいていこれで十分
です。

**問題が起きたとき**は、代わりに失敗の通知が送られます。処理が止まったステージ
とエラー内容が記載されるので、フォルダのオーナーが原因を調べられます。この通知
が届いた場合や、書類がいつまでも処理されない場合は、[トラブルシューティングと
FAQ](./troubleshooting.md)を参照してください。
