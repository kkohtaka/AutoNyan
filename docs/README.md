# AutoNyan user documentation

Bilingual (English / 日本語) end-user documentation site, built with
[Docusaurus](https://docusaurus.io/) and published to GitHub Pages at
<https://kkohtaka.github.io/AutoNyan/>.

This is a **standalone** npm project — it is intentionally not part of the root
npm workspace, so the Cloud Functions CI never installs the Docusaurus
toolchain. Run all commands from this `docs/` directory.

## Local development

```bash
cd docs
npm ci
npm start            # dev server with hot reload (default locale)
npm start -- --locale ja   # preview the Japanese site
```

## Build

```bash
cd docs
npm ci
npm run build        # outputs static site to docs/build/
npm run serve        # serve the built site locally
```

Publishing happens automatically: pushing to `master` runs
`.github/workflows/docs.yml`, which builds this site and deploys it to GitHub
Pages.

## Information architecture

The user docs are grouped into top-level sections, shown in the sidebar in this
order:

1. **Welcome** — the landing page (`docs/index.md`, served at `/`).
2. **Getting Started** (`docs/getting-started.md`) — what you need and how to
   add your first document.
3. **Daily Use** (`docs/daily-use.md`) — the everyday Drive intake →
   classification → notification-email flow.
4. **Troubleshooting & FAQ** (`docs/troubleshooting.md`).

The sidebar is autogenerated from the `docs/` tree. Order comes from each page's
`sidebar_position` front matter, and the sidebar label comes from the page
`title` — so labels are translated automatically by the Japanese page's `title`.
Each section is currently a **single Markdown file**, not a folder. Adding a page
is just dropping a Markdown file in — no `sidebars.ts` edit needed.

> **Why flat files, not folders with `_category_.json`?** A folder whose only doc
> is an `index.md` used as the category `link` collapses to a single sidebar
> entry that always shows the **default-locale** category label, so the Japanese
> sidebar would show English. Flat pages avoid that because they are labelled by
> their (localized) `title`. Only introduce a `_category_.json` once a section
> actually has multiple child pages (see below) — at that point the category
> renders normally and its label localizes.

Keep user docs conceptual: describe what to do and what to expect, not the
implementation. Implementation details belong in the
[developer documentation](https://github.com/kkohtaka/AutoNyan#readme).

## How to add a bilingual page

Every page exists twice: the English source and its Japanese translation at the
mirrored path. To add one page:

1. **English (default locale):** create the Markdown file under `docs/`, e.g.
   `docs/getting-started.md`. Set its front matter (`title`, `sidebar_position`).
2. **Japanese:** create the same relative path under
   `i18n/ja/docusaurus-plugin-content-docs/current/`, e.g.
   `i18n/ja/docusaurus-plugin-content-docs/current/getting-started.md`. Translate
   the body and the `title`. The `slug` and `sidebar_position`, if any, must
   match the English file.
3. Verify both locales build: `npm run build` (it fails on broken links) and,
   optionally, preview with `npm start -- --locale ja`.

If a Japanese page is missing, Docusaurus falls back to the English content for
that page, so the site still builds — but untranslated pages should be treated
as a gap to fill.

### Growing a section into multiple pages

When a section needs more than one page, turn its single file into a folder:

1. Move `docs/<section>.md` → `docs/<section>/index.md` (and the same for the
   Japanese file under `i18n/ja/...`).
2. Add the additional pages as Markdown files in that folder, in both locales.
3. Add a `docs/<section>/_category_.json` with the section `label` and
   `position`, and `link: { type: "doc", id: "<section>/index" }` so the section
   overview opens when the category is clicked.
4. Run `npm run write-translations -- --locale ja`. This creates a localized
   `i18n/ja/docusaurus-plugin-content-docs/current/<section>/_category_.json`;
   translate its `label` into Japanese. (Now that the folder has real child
   pages, the category renders as a collapsible group and the Japanese label is
   used.)
5. `npm run build` to confirm both locales.

UI string translations (navbar, footer, theme) live under
`i18n/ja/docusaurus-theme-classic/`; regenerate their templates with
`npm run write-translations -- --locale ja` whenever navbar/footer items change.
